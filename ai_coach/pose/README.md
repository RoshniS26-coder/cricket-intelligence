# Pose Analysis

MediaPipe Tasks API pose extraction → batsman technique features → narrated
annotated video. Side-on camera angle is required for usable features.

Library code: `src/pose/extractor.py`, `src/pose/smoothing.py`,
`src/pose/features/batsman.py`, `src/report/video_renderer.py`,
`src/report/tts.py`, `src/report/mux.py`.

**Pre-req:** MediaPipe needs Python 3.12 — activate `venv312` before
running.

## Quick recipe

| Scenario | Recipe |
|---|---|
| Render annotated narrated MP4 for one ball clip | `source venv312/bin/activate && features/pose_analysis/run_render.sh <clip> "<player>"` |

The recipe sanity-checks that you're on Python 3.12 and exits early if not.

## Command

### `render_ball_video.py` — clip → annotated narrated MP4

Steps: Gemini extraction (optional) → MediaPipe pose → smoothing →
batsman features (head offset, stride, shoulder angle) → narration text →
Edge TTS → OpenCV overlay → ffmpeg slowdown → ffmpeg mux.

```bash
source venv312/bin/activate
python features/pose_analysis/render_ball_video.py [options]
```

| Flag | Default | Purpose |
|---|---|---|
| `--clip PATH` | required | Single-ball video |
| `--player NAME` | `Player A` | Shown in the overlay |
| `--out PATH` | `data/reports/videos/ball_demo.mp4` | Output MP4 |
| `--slowdown FLOAT` | `2.0` | Base slowdown factor (ffmpeg `setpts`) |
| `--no-gemini` | off | Skip Gemini extraction; pose-only narration |
| `--voice NAME` | `en-IN-PrabhatNeural` | Edge TTS voice |

The narration is generated by Gemini if `GEMINI_API_KEY` is set; otherwise
a template-based fallback is used (still cites real numbers from the pose
features).

## Output

- `<out>` — annotated MP4 (overlay + narration audio muxed in, video
  stretched to match audio length)
- `<out>.mp3` — Edge TTS narration audio (kept alongside)

## Example

```bash
source venv312/bin/activate
python features/pose_analysis/render_ball_video.py \
    --clip data/raw_videos/kohli-nets-20260506.mp4 \
    --player "Virat Kohli" \
    --out data/reports/videos/kohli_demo.mp4
```

## Feature outputs (from `src/pose/features/batsman.py`)

| Feature | Target | Meaning |
|---|---|---|
| `head_lateral_offset` | `< 0.03` | Head should be over the ball at impact |
| `head_over_ball` | `True` | Boolean derived from offset |
| `stride_length_norm` | `> 0.35` | Normalized stride length (front foot vs hip width) |
| `stride_adequate` | `True` | Boolean derived from stride |
| `shoulder_angle_deg` | side-on ~90° | Body shape at impact |
